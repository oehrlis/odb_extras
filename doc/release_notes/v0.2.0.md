# OraDBA Extras Extension v0.2.0

**Release Date:** 2026-02-17

## Overview

Version 0.2.0 introduces optional OraDBA etc hook support scaffolding for
environment and aliases, with safe defaults that keep the hooks disabled.

## Highlights

- Added `.extension` flags:
  - `load_env: false`
  - `load_aliases: false`
- Added hook files:
  - `etc/env.sh`
  - `etc/aliases.sh`
- Added documentation for enabling and using extension etc hooks.

## Details

### Added

- Extension metadata
  - `.extension` now declares `load_env` and `load_aliases` explicitly.
  - Both are intentionally set to `false` by default.
- `etc/env.sh`
  - Optional environment hook for extension-local exports and PATH adjustments.
  - Designed to be idempotent when sourced multiple times.
- `etc/aliases.sh`
  - Optional alias hook for convenience commands.
  - Loaded only when alias hook loading is enabled.

## How to Enable

To activate hooks for this extension, set both conditions:

1. Global OraDBA switch:

```bash
export ORADBA_EXTENSIONS_SOURCE_ETC=true
```

1. Extension metadata flags in `.extension`:

```yaml
load_env: true
load_aliases: true
```
